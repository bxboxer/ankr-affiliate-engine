# ANKR Affiliate Engine

## Project Identity
This is the central dashboard and automation engine for managing a network of affiliate review sites. It deploys new sites, scores content, gathers competitive intelligence, and tracks performance — all from one place.

## Tech Stack
- **Framework**: Next.js 15 (App Router) + React 19 + TypeScript
- **Styling**: Tailwind CSS 4 + DaisyUI 5 (dark theme)
- **Database**: Vercel Postgres + Kysely (type-safe SQL)
- **Auth**: Simple admin password gate (no Auth0 — this is single-user admin only)
- **AI**: Anthropic Claude API via @anthropic-ai/sdk
- **Hosting**: Vercel (blaine-personal account)
- **GitHub**: bxboxer account (affiliate sites separated from main ANKR repos)

## Architecture
```
src/app/           → Next.js pages and API routes
src/lib/db/        → Kysely database schema, migrations, connection
src/lib/auth.ts    → Session-based admin auth
src/components/    → Shared React components
orchestrator/      → Agent scripts (spawner, scorer, recon)
.github/workflows/ → GitHub Actions for automated runs
```

## Key Conventions
- Match ANKR Agency Marketing conventions where possible (same dev)
- Use DaisyUI component classes (`btn`, `card`, `badge`, `table`, etc.)
- API routes return JSON with consistent `{ error, detail }` shape on failure
- Zod validation on all POST endpoints
- Database uses UUID primary keys with `gen_random_uuid()`
- All timestamps are `timestamptz` with `now()` defaults

## Affiliate Network
- Sites deploy to Vercel under blaine-personal
- GitHub repos under bxboxer account, prefixed with `ankr-`
- Each site gets its own CLAUDE.md generated by the spawn agent
- Amazon Associates is the primary program, but sites should diversify

## STOP AND ASK
- Before deleting any site or its data
- Before changing database schema
- Before modifying GitHub Actions workflow triggers
- Before any action that would affect production deployments
